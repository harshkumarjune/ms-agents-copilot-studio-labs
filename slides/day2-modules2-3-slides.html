Copilot Studio was formerly Power Virtual Agents. It's evolved from simple chatbots to a full agent development platform. Today it supports MCP, bring your own model, and enterprise governance.

When to Use Pro-Code in Copilot Studio

Use Pro-Code When:

  • Custom API integrations needed
  • Advanced orchestration logic
  • Integrating custom AI models
  • Complex data transformations
  • External MCP servers

Stay Low-Code When:

  • Simple Q&A scenarios
  • Standard connectors sufficient
  • Topic-based conversations
  • Built-in capabilities work
  • Rapid prototyping
Pro-code gives you power but adds complexity. Use it when built-in capabilities aren't enough. Most agents can start low-code and add pro-code components as needed.

MCP enables agents to access custom tools, data sources, and prompt templates.

MCP is a standardized protocol for connecting AI agents to external tools and data. It uses JSON-RPC 2.0 and can run over stdio or HTTP. This is how you give Copilot Studio agents access to custom capabilities.

MCP Server Components

Tools

Functions agents can call

tools/list tools/call

Execute actions,
return results

Resources

Data sources to read

resources/list resources/read

Read-only data,
schemas, docs

Prompts

Reusable templates

prompts/list prompts/get

Standardized prompts,
with arguments

MCP servers expose three types of capabilities. Tools are functions the agent can call. Resources are data sources to read. Prompts are reusable templates.

MCP Server Implementation

import { Server } from "@modelcontextprotocol/sdk/server"; const server = new Server({ name: "my-server", version: "1.0.0" }, { capabilities: { tools: {}, resources: {} } }); // Handle tool calls server.setRequestHandler(CallToolRequestSchema, async (req) => { const { name, arguments: args } = req.params; // Execute tool logic... return { content: [{ type: "text", text: result }] }; });
Here's the basic structure of an MCP server. You create a server instance, define capabilities, and set up request handlers for tools, resources, and prompts.

Agent Cost Controls

What You Can Control

  • Monthly message limits
  • Per-user quotas
  • API call limits
  • Storage limits
  • Alert thresholds

Best Practices

  • Start with conservative limits
  • Monitor usage patterns
  • Set alerts at 80% usage
  • Review monthly reports
  • Adjust based on value delivered
Cost controls prevent runaway spending on AI agents. Set monthly limits, per-user quotas, and alert thresholds. Start conservative and adjust based on actual usage and value.
