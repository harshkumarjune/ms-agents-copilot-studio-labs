Microsoft 365 Copilot was designed from day one to be extended. Choose your extensibility path based on requirements.

There are three main ways to extend Microsoft 365 Copilot. Declarative Agents are the fastest path - mostly configuration. Message Extensions are for interactive cards and search. Custom Engine Agents give full code control for advanced scenarios.

Declarative Agent Architecture

Defined by JSON manifest - you describe what, the platform handles how.

// declarativeAgent.json { "name": "HR Assistant", "instructions": "You help employees with HR questions...", "actions": [...] }

Manifest Components

  • Name & Description
  • Instructions (system prompt)
  • Capabilities
  • Actions (plugins/APIs)
  • Conversation Starters
A declarative agent is defined by a JSON manifest. You don't write orchestration code. The instructions field is where the magic happens - it's your chance to program the AI's personality, knowledge boundaries, and behavior patterns.

When to Choose Custom Engine

Use Custom Engine When:

  • Complex multi-step reasoning required
  • Need to integrate custom/fine-tuned LLM
  • Custom tool orchestration logic needed
  • Persistent memory across sessions
  • Advanced error handling required

TypeSpec for API Definitions

Type-safe language that compiles to OpenAPI. Catches errors at compile time, not runtime.

@route("/repairs") interface RepairOperations { @get list(): Repair[]; @post create(@body request: RepairRequest): Repair; }

Benefits

  • Type safety
  • Compiles to OpenAPI
  • Cleaner than raw JSON
  • IDE support
  • Validation at compile time
TypeSpec is a language developed by Microsoft for defining API contracts. It's type-safe, compiles to OpenAPI, and catches errors before runtime. This is much cleaner than writing raw OpenAPI JSON.

Adding Actions with Plugins

Plugins wrap APIs to make them understandable to the AI. The description is critical - the AI uses it to decide when to call your function.

{ "functions": [{ "name": "getRepairStatus", "description": "Get the current status of a repair ticket", "parameters": { "type": "object", "properties": { "ticketId": { "type": "string" } } } }] }
Plugins let agents DO things. A plugin is a wrapper around an API that makes it understandable to the AI. You define what the plugin does, what inputs it needs, and what it returns. The description is critical - be specific and clear.

Semantic Kernel

Microsoft's AI orchestration SDK - the backbone of your custom agent.

  • Plugin management
  • Memory & context handling
  • Planners for complex tasks
  • Custom orchestration
const kernel = new Kernel(); // Add plugins kernel.addPlugin( new SearchPlugin(), "search" ); // Invoke prompt const response = await kernel .invokePrompt("Summarize: {{$input}}");
Semantic Kernel is Microsoft's AI orchestration SDK. Think of it as the backbone of your custom agent. It provides plugin management, memory handling, planners for complex tasks, and custom orchestration.
